Projets

Projet principal

Manaketak

Manaketak est une application web de gestion de commandes de photos scolaires. J’ai conçu et développé ce projet en autonomie, de l’analyse du besoin à la mise en production, avec une architecture full-stack et un déploiement sur Scalingo, Vercel et OVH.

Contexte

L’objectif était de simplifier la gestion des commandes de photos de classe, avec plusieurs types d’utilisateurs (parents, écoles, administrateur) et une logique métier spécifique.

Réalisation

  • conception de l’architecture et de la base de données
  • développement d’une API REST sécurisée (Symfony, API Platform)
  • gestion des rôles utilisateurs
  • développement du frontend (Next.js)
  • gestion des commandes
  • génération de PDF, exports Excel et e-mails

Déploiement

  • backend déployé sur Scalingo
  • frontend déployé sur Vercel
  • stockage des fichiers via OVH Object Storage
  • configuration des variables d’environnement

Architecture

L’application repose sur une architecture frontend / backend, avec un stockage externe pour les fichiers.

Le frontend communique avec l’API backend qui gère les données et le stockage.

Démo de l'application

Deux démonstrations courtes montrent les usages côté administrateur et côté parent.

Interface administrateur

Gestion des écoles, des classes et des commandes depuis l’espace admin.

Parcours parent

Consultation des photos et passage d’une commande en quelques étapes.

Test technique

Application de blagues – Carambar

Réalisation d’un test technique consistant à développer une mini application web de blagues, avec une API et une interface utilisateur simple permettant d’afficher une blague aléatoire.

Contexte

Ce projet a été réalisé dans le cadre d’un test de sélection. L’objectif était de répondre à un cahier des charges imposé, incluant la création d’une API versionnée et d’une interface frontend.

Réalisation

  • développement d’une API REST (Node.js, Express, Sequelize, SQLite)
  • création de plusieurs endpoints (liste, détail, ajout, aléatoire)
  • mise en place d’une interface simple avec un bouton affichant une blague
  • documentation de l’API avec Swagger
  • déploiement du backend (Render) et du frontend (GitHub Pages)

Interface

Ce que ce projet démontre

  • capacité à comprendre et appliquer un cahier des charges
  • mise en place d’une API structurée
  • connexion frontend / backend
  • organisation du code et des livrables
  • autonomie dans un contexte contraint

Tester l’API

En raison de l’hébergement gratuit, la base peut être réinitialisée. Vous pouvez tester l’API en ajoutant une blague via Swagger, par exemple :

  • « Que dit un oignon quand il se cogne ? Aïe »
  • « Quel est l’animal le plus heureux ? Le hibou, parce que sa femme est chouette. »
